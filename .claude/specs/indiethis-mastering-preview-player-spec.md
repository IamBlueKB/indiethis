## PLAYER DESIGN

The compare screen gets a new player that replaces the current basic audio player. The design uses IndieThis branding elements for a unique, premium feel.

### Layout (top to bottom)

1. **Logo-shaped player** — rounded square outline (matching IndieThis icon corner radius) with coral play/pause button centered inside and a gold progress sweep around the edges
2. **Track info** — version name (e.g. "Warm"), "mastered"/"unmastered" label, time counter (0:00 / 0:30)
3. **Reference note** — small text with a gold dot: "Reference: {filename}". Only visible when `referenceTrackUrl` exists on the job. Hidden otherwise.
4. **A/B toggle** — "Original" label, toggle switch, "Mastered" label. Active side highlighted in gold.
5. **Dual waveform display** — standalone dark box showing two overlapping waveforms
6. **Version boxes** — 4 separate cards in a row (Clean / Warm / Punch / Loud)
7. **Console stats** — LUFS, Peak, Range, Width with animated meter bars
8. _(existing download/export section below — do not modify)_

## 1. LOGO-SHAPED PLAYER

- Rounded square outline matching the IndieThis icon shape (large corner radius ~30% of width)
- Outline stroke: 6px, `var(--color-border-tertiary)` at 12-15% opacity for the background track
- Gold progress sweep: 6px stroke, `#D4AF37`, `stroke-linecap: round`
- **Progress starts at top center** of the rounded square and sweeps clockwise
- Use SVG `<path>` for the rounded square, calculate total path length with `getTotalLength()`, animate via `stroke-dashoffset`
- Coral play button centered inside: triangle pointing right, fill `#E8735A`, ~48px wide
- Coral pause button: two vertical bars, same coral color, same size
- Play button has a subtle pulse animation (`box-shadow` keyframes) when idle, stops when playing
- Click anywhere on the rounded square outline to scrub — calculate angle from center using `atan2` with origin at top center (not left), map to 0-30 seconds
- The player plays the 30-second preview clip (highest energy section, already generated by the `preview` action)

## 4. A/B TOGGLE

- "Original" text left, toggle switch center, "Mastered" text right
- Active side text: `#D4AF37`, weight 500
- Inactive side text: tertiary text color
- Toggle track: 44px wide, 24px tall, rounded pill, gold-tinted background
- Toggle thumb: 18px circle, `#D4AF37`, slides left/right with 0.2s transition
- Clicking either label also toggles (not just the switch)
- When toggled to "Original":
  - Version name changes to "Original", label changes to "unmastered"
  - Version boxes dim to 30% opacity and become non-interactive (`pointer-events: none`)
  - Console stats update to show original track values
  - Waveform shows only the original (grey) waveform, gold mastered waveform hidden
  - **Audio playback switches to original preview clip**
- When toggled back to "Mastered":
  - Version boxes restore to full opacity and interactive
  - Previously selected version re-activates
  - Waveform shows both original and mastered overlays
  - **Audio playback switches to the selected version's preview clip**

### Volume Matching (CRITICAL)

- When toggling between Original and Mastered, normalize playback volume so both sound equally loud
- Use the `inputLufs` (from analysis) and the selected version's LUFS to calculate gain offset
- Apply gain offset to the `<audio>` element's volume: `audio.volume = Math.min(1, 10 ** ((targetLufs - versionLufs) / 20))`
- This prevents the "louder = better" bias — artist hears the real tonal difference, not just volume
- Without this, artists will always prefer "Loud" because it's loudest, not because it actually sounds best

## 5. DUAL WAVEFORM DISPLAY

- Standalone dark box: `rgba(10,10,10,0.5)` background, 0.5px border, 14px border-radius
- Full width (max-width 380px), height 120px
- Canvas element renders two waveforms:
  - **Original waveform**: grey/white at ~15% opacity
  - **Mastered waveform**: gold (`#D4AF37`) at ~60% opacity, with subtle gold fill at ~6% between waveform and center line
- Waveforms are mirrored top and bottom from center line
- Gold playhead line (2px, with subtle glow) syncs with the logo-shaped player above — both show the same position
- Click anywhere on the waveform box to scrub (syncs both waveform playhead and logo sweep)
- Top-left corner legend: two small dots with labels — grey dot "Original", gold dot "{VersionName}"

### Real Waveform Data (IMPORTANT)

- Do NOT use randomly generated waveforms
- Generate waveform data from the actual audio files server-side:
  - When the mastering job completes, extract waveform peak data from the preview clips using `librosa` or `ffmpeg`
  - Store as a JSON array of normalized amplitude values (0-1) on the job record — field `previewWaveform` (Json, array of ~200 float values)
  - For the mastered version, store per-version waveforms: `cleanWaveform`, `warmWaveform`, `punchWaveform`, `loudWaveform` (or a single `versionWaveforms` Json object)
- The compare screen fetches these arrays and renders them on the canvas
- If waveform data is not yet available (legacy jobs), fall back to a flat line at 50% amplitude

### Waveform Generation (Python — add to predict.py)

Add a helper that runs after each version is created in `_master`:

```python
def extract_waveform(audio_path, num_points=200):
    y, sr = librosa.load(audio_path, sr=None, mono=True)
    # Divide into num_points chunks, take max amplitude per chunk
    chunk_size = len(y) // num_points
    peaks = []
    for i in range(num_points):
        chunk = y[i * chunk_size : (i + 1) * chunk_size]
        peaks.append(float(np.max(np.abs(chunk))))
    # Normalize to 0-1
    max_peak = max(peaks) if max(peaks) > 0 else 1
    return [p / max_peak for p in peaks]
```

Return waveform data alongside the version URLs in the master action output. Store on the job record via the master webhook.

## 6. VERSION BOXES

- 4 separate cards in a single row, equal width, 8px gap
- Each card: 14px vertical padding, 12px border-radius, 0.5px border
- Default state: `var(--color-border-tertiary)` border, `rgba(255,255,255,0.02)` background
- Hover state: gold-tinted border at 30% opacity, gold-tinted background at 3%
- Active state: gold border at 50% opacity, gold background at 8%, plus a 2px gold accent line at the bottom (centered, 60% width)
- Card content:
  - Version name: 13px, weight 500. Default: secondary text color. Active: `#D4AF37`
  - Description: 9px, tertiary text color, 1 line
    - Clean: "Transparent, natural"
    - Warm: "Rich low end, smooth"
    - Punch: "Mid presence, tight"
    - Loud: "Maximized, bold"
- No LUFS values in the cards — stats are in the console below
- Clicking a card switches audio playback to that version's preview, updates console stats, updates waveform to show that version's waveform data

### Mobile Layout (screens < 640px)

- Version boxes switch from 4-across to 2x2 grid: `grid-template-columns: 1fr 1fr`
- Gap stays 8px
- Everything else stays the same layout — logo player, waveform, console all remain full width

## 7. CONSOLE STATS

- Full width (max-width 380px), single container with subtle background and border, 14px border-radius
- 4 meters in a row separated by thin 0.5px vertical dividers
- Each meter:
  - Value: 18px, weight 500, mono font, primary text color
  - Meter bar: 3px height, full width, tertiary background, colored fill with 0.4s transition on width
  - Label: 9px, uppercase, 1px letter-spacing, tertiary text color
- Meters:
  - **LUFS**: value from selected version, gold highlight when mastered
  - **Peak**: true peak dB value
  - **Range**: dynamic range value, gold highlight when mastered
  - **Width**: stereo width percentage
- When A/B toggled to Original, all values update to original track analysis. Gold highlights removed.
- When version changes, values animate to new numbers, meter bar widths transition smoothly

### Stats Data Source

- Original values: from `inputLufs`, `inputBalance` on the job record
- Version values: from the master action output — each version returns `{ lufs, truePeak, dynamicRange, stereoWidth }`
- If per-version stats aren't available yet (requires predict.py update + cog push), show the values from the version's LUFS measurement that Sonnet already added

## 8. AUDIO PLAYBACK

- The player plays the 30-second preview clips, NOT the full mastered files
- Preview clips are already generated by the `preview` action in the pipeline
- For the original, play from the original uploaded file (use `inputFileUrl`) — but only the same 30-second window
- For versions, each version needs its own 30-second preview clip generated from the same time window
- If per-version previews don't exist yet, play from the full version file but seek to the preview start position

### Preview Clip Strategy

The current pipeline generates one preview clip. For the compare screen to work properly with version switching, the engine should generate preview clips for each version from the same 30-second window:

- `_preview` runs once to find the best 30-second window → returns `start_sample` position
- `_master` uses that same window to generate per-version preview clips: `preview_clean.wav`, `preview_warm.wav`, `preview_punch.wav`, `preview_loud.wav`
- Plus `preview_original.wav` from the unprocessed input at the same window
- All 5 preview clips stored in Supabase, file paths on the job record

This ensures the A/B comparison is apples-to-apples — same section of the song, just different processing.

**This requires updating predict.py and doing a cog push.** Can be done in phases:
- Phase 1: Use the single existing preview clip for all versions (works but not ideal)
- Phase 2: Generate per-version previews from the same window (full experience)

## IMPLEMENTATION NOTES

- This replaces the current compare screen audio player only — do not touch the download section, format selector, or any other post-compare UI
- The logo-shaped SVG path must use `getTotalLength()` at runtime to calculate the correct `stroke-dasharray` and `stroke-dashoffset` values
- Scrub on both the logo shape and waveform must stay synced — clicking either updates both positions and the audio `currentTime`
- The `<audio>` element should be a single hidden element. Switching versions changes `audio.src`. Switching A/B changes `audio.src` and applies volume normalization.
- Preserve playback position when switching versions or toggling A/B — don't restart from 0:00
- All colors use the IndieThis palette: gold `#D4AF37`, coral `#E8735A`, dark backgrounds, tertiary borders
